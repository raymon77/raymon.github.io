关于Angular的十问十答(一)

  • Anguluar

关于Angular的十问十答系列，意在梳理Angular框架相关的知识架构，希望能对这部分知识形成一个完整的回路有所帮助。至于本系列具体会写几篇，我也还不确定，毕竟是从头开始梳理，但是难度系数肯定是逐渐增强的。前两篇都是入门级，不过更新仍然不一定按照顺序来，想到哪了，就写到哪儿，就这么任性。

1.什么是MVC? 使用MVC的优点是什么？

1.1 MVC是一种软件架构设计的思想，将软件按照模型、视图、控制器来划分。模型负责封装业务逻辑和数据；视图负责输入和输出，控制显示界面交互；控制器负责协调模型和视图。

  • Model(模型)：模型是应用程序的主体部分，通常用于处理应用程序的业务数据，或者业务逻辑部分.
  • View(视图)：视图是应用程序中处理用户界面显示的部分，通常视图是依据模型数据创建的.
  • controller(控制器)：应用程序中处理用户交互的部分，通常控制器负责从视图读取数据，控制用户输入，并向模型发送数据。
  • 1.2 MVC的优点：
    • 实现了功能模块和显示模块的分离
    • 可维护性
    • 可扩展性
    • 可移植性和组件的可复用性

      2.什么是指令？

      HTML在构建应用（App）时存在诸多不足之处，AngularJS通过扩展一系列的HTML属性或标签来弥补这些缺陷，所谓指令就是AngularJS自定义的HTML属性或标签，这些指令都是以ng-做为前缀的，例如ng-app、ng-controller、ng-repeat等。

      3.列举出一些常见的ng当中内置指令？它们的作用分别是什么？

  • ng-app 指定应用根元素，至少有一个元素指定了此属性。
  • ng-controller:指定控制器对象
  • ng-show:控制元素是否显示，true显示、false不显示
  • ng-hide:控制元素是否隐藏，true隐藏、false不隐藏
  • ng-if:控制元素是否“存在”，true存在、false不存在
  • ng-src:指定 <img> 元素的 src 属性,增强图片路径
  • ng-href:为 <a> 元素指定链接,增强地址
  • ng-class:指定HTML元素使用的 CSS 类，控制类名
  • ng-include:在应用中包含HTML文件，引入模板
  • ng-disabled:规定一个元素是否被禁用，表单禁用
  • ng-readonly:指定元素的readonly属性，表单只读
  • ng-checked:规定单/复选框表单是否被选中
  • ng-selected:规定下拉框表单是否被选中

    4.什么是数据的双向绑定？什么时候使用双向绑定？

  • 数据双向绑定:视图的数据可以绑定到模型当中,模型的数据可以绑定到视图当中，不仅能读取数据，也能更新数据.
  • 使用场景：表单编辑/交互等.

7.列出directive当中restrict的几种形式，分别什么意思？

  • 自定义指令可以以4中形式展示到HTML中：简称‘ECMA’
  • E:element：以元素/标签的形式使用
  • C:以标签的类名形式使用
  • M:以注释形式出现和使用
  • A:attribute:以属性的形式使用

    8.指令当中transclude代表什么意思？

    规定填充的目标位置

    9.什么是过滤器？列举你所知道的内置过滤器？

    过滤器可以用来转换数据，格式化数据。过滤器可以使用一个管道字符（|）添加到表达式和指令中。
    angular内置了9个过滤器：
  • 1.Currency：将数据格式化为货币格式
  • 2.Date:日期格式化，年（y）、月（M）、日（d）、星期（EEEE/EEE）、时（H/h）、分（m）、秒（s）、毫秒（.sss），也可以组合到一起使用。
  • 3.Filter：从数组项中选择一个子集。
  • 4.Json:将Javascrip对象转成JSON字符串。
  • 5.limitTo:取出字符串或数组的前（正数）几位或后（负数）几位
  • 6.Lowercase：格式化字符串为小写。
  • 7.Uppercase：格式化字符串为大写。
  • 8.Number：数字格式化，可控制小数位数
  • 9.orderBy：根据某个表达式排列数组。
